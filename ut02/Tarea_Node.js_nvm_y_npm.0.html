Node.js

¿Qué es Node.js?

Node.js es un entorno en tiempo de ejecución multiplataforma, de código abierto, para la capa del servidor (pero no limitándose a ello) basado en el lenguaje de programación JavaScript, asíncrono, con E/S de datos en una arquitectura orientada a eventos y basado en el motor V8 de Google. Fue creado con el enfoque de ser útil en la creación de programas de red altamente escalables, como por ejemplo, servidores web.

Node.js también nos permite practicar la programación Javascript sin la necesidad e utilizar el motor del navegador para la ejeción del código.

Tambiés es necesario disponer de Node.js para la instalación del gestor de paquetes NPM, nos va a facilitar la instalación de utilidades Front-End en el sistema.

Instalación

Se puede realizar descargando de su web en la dirección https://nodejs.org

Pero es mejor usar la herramienta NVM ya que nos permitirá lidiar con los problemas de versiones que aparecen en Node.js ya que al ser una herramienta código abierto tiene diferentes versiones con diferentes tipos de soporte

LTS. Long Term Support Soporte de Largo Plazo, versión que se asegura estabilidad y que dispondrá de actualizaciones durante un tiempo muy prolongado.

CurrentActual. Última versión pero no se considera estable. Cada cierto tiempo la Current pasa a Active

NVM

¿Qué es NVM?

Node Version Manager (nvm), permite instalar diferentes versiones de nodejs sin tener conflicto entre ellas.

Instalación

Para instalar o actualizar, se debe ejecutar el script install.sh disponible en https://github.com/nvm-sh/nvm o directamente ejecutando cualqueira de estos dos comandos.

curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.35.3/install.sh | bash
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.35.3/install.sh | bash

Comandos básicos de NVM:

nvm install node

Si se desea instalar alguna versión especifica de Node:

nvm install 10.16.0

Para conocer todas las versiones de Node disponible:

nvm ls-remote

Para conocer la versión de Node actual en uso:

nvm current

Para saber que versiones de Node hay instaladas:

nvm ls

Si se desea ejecutar algún script con alguna versión especifica de Node:

nvm exec 10.10.0 node script.js

Para conocer el path de una versión de Node instalada:

nvm which 10.10.0

Si se desea desinstalar una versión de Node:

nvm uninstall 10.10.0

NPM

¿Qué es NPM?

npm es el sistema de gestión de paquetes por defecto para Node.js, un entorno de ejecución para JavaScript, bajo Artistic License 2.0.

Desde la versión 0.6.3 de Node.js npm es instalado automáticamente con el entorno. npm se ejecuta desde la línea de comandos y maneja las dependencias para una aplicación. Además, permite a los usuarios instalar aplicaciones Node.js que se encuentran en el repositorio.

Tambiés es necesario disponer de Node.js para la instalación del gestor de paquetes NPM, nos va a facilitar la instalación de utilidades Front-End en el sistema.

Los paquetes que instalemos lo podemos hacer de dos formas en el directorio del proyecto actual(donde se ejecuta el comando) o con el parámetro -g, de esta última forma estará disponible en cualquier parte que se necesite.

Presentamos un listado de Comandos básicos para nuevos usuarios de NPM.

Inicializar el proyecto crea el archivo package.json

npm init

Buscar paquetes

npm search [nombreDelPaquete]

Instalar un paquete

npm install [nombreDelPaquete]

Instalar una version especifica de un paquete usando “version” irá la versión especificada, de lo contrario, si no se especifica, se instalará la última disponible.

npm install nombreDelPaquete@version

Instalar el paquete y modifica el archivo package.json agregando la librería en las dependencias

npm install [nombreDelPaquete] --save

Instalar el paquete y modifica el archivo package.json agregando la librería en las dependencias solo necesarias para desarrollo

npm install [nombreDelPaquete] --save-dev

Si no se especifica un paquete, se entiende que se desea verificar/instalar todas las dependencias dentro del archivo package.json. Esto es habitual, cuando se descargan proyectos o aplicaciones de g ithub, ya que las dependencias deben ser instaladas luego de descargar el proyecto (por razones de tamaño).

npm install

Con la opción -g o –global se le está indicando a npm que debe instalar el paquete de manera Global, esto quiere decir que se va a poder utilizar desde cualquier directorio.

npm install -g [nombreDelPaquete]

Eliminar paquete local

npm rm [nombreDelPaquete]

npm uninstall [nombreDelPaquete]

Eliminar paquete de forma global

npm rm -g [nombreDelPaquete]

npm uninstall -g [nombreDelPaquete]

Listar las dependencias de mi proyecto

npm ls

Consultar las últimas versiones compatibles con las librerías que tienes en tu proyecto

npm outdated
Actualizar todas las librerías de tu proyecto que sean compatibles

npm update -save

¿Qué es Git?

Git (pronunciado "guit"2​) es un software de control de versiones diseñado por Linus Torvalds, pensando en la eficiencia y la confiabilidad del mantenimiento de versiones de aplicaciones cuando éstas tienen un gran número de archivos de código fuente. Su propósito es llevar registro de los cambios en archivos de computadora y coordinar el trabajo que varias personas realizan sobre archivos compartidos.

Vamos a tener dos formas de utilizar Git una de forma local y remota.

En forma local lo que hacemos es iniciar un directorio para un proyecto. Cuando lo consideremos necesario iremos haciendo commint de esta forma si tengo problemas con alguna modificación podré volver a un estado anterior. También puedo llevar varias ramas de desarrollo

En modo remoto la diferencia es que cuando relizo los commint también los puedo subir a un repositorio, donde otros pueden sincronizar, descargar, ...

Instalar Git

sudo apt install git

Configurar Git

Primero crearemos una cuenta en https://github.com

Si utilizamos la opción --global, esto solo lo tenemos que hacer una vez, puesto que a partir de entonces lo tomará por defecto o sin esa opción en cada proyecto (directorio) ejecutamos el comando sin --golbal . Así por ejemplo,

git config --global user.name "RicardoUsuarioGit"
git config --global user.email Usuario@mi_correo.es

Configurar Git

Creamos un directorio

mkdir proyecto1
cd proyecto1
git init
Si vemos el contenido del directorio encontramos que se ha creado un directorio oculto llamado .git donde se encuentran los archivos necesarios para guardar las versiones.

Si lo que queremos es descargar un repositorio de la nuve en el directorio utilizamos:

git clone https://github.com/RicardoUsuarioGit/proyecto1.git
Nos descarga los ficheros del proyecto en el directorio donde nos encontramos.

Añadir ficheros al proyecto

Los únicos ficheros que se tendrán en cuenta para al hacer un commit serán los indicados con el comando:

git add nombre_fichero
git add . (para todos los ficheros)
Solo se procesa los que se modificarán desde el anterior commit

Guardar los ficheros modificados

git commit -m "Maquetación hecha"

Publicar en el repositorio actual

git push rama

Descargar el contenido del repositorio remoto y mezclar con el actual

git pull rama
